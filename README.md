# MetalPulse

Live precious metals spot prices, charts, and price alerts.

🚀 **Live demo**: https://metalpulse-production-941e.up.railway.app

## Features

- 📈 Live spot prices for Gold (XAU), Silver (XAG), Platinum (XPT), Palladium (XPD)
- 📊 30‑day historical charts (Stooq daily closes)
- 📱 Mobile‑friendly & installable PWA
- 📧 Email price alerts (crossing‑based, 15‑minute checks)
- 📊 Popular metal ETFs with live quotes
- 💱 USD/EUR currency toggle
- 🌙 Dark/light theme

## Tech Stack

- **Frontend**: React + TypeScript + TailwindCSS + shadcn/ui
- **Backend**: Node.js + HTTP server (no Express)
- **Data sources**:
  - Live spot: Stooq (primary) → TwelveData (fallback)
  - Historical: Stooq daily closes
  - ETFs: TwelveData
- **Email**: SendGrid (no-reply@sendgrid.net or verified sender)
- **Database**: Supabase
- **Deployment**: Railway (API + Worker services)

## Quick Start

### Prerequisites

- Node.js 20+
- npm

### Install

```bash
git clone <repo-url>
cd metalpulse
npm install
```

### Environment Variables

Create a `.env` file in the project root:

```env
# Supabase (backend DB)
SUPABASE_URL=your_supabase_url
SUPABASE_SERVICE_ROLE_KEY=your_service_role_key

# TwelveData (live spot + ETFs)
TWELVEDATA_API_KEY=your_twelvedata_key

# SendGrid (alerts)
SENDGRID_API_KEY=your_sendgrid_api_key
ALERT_FROM_EMAIL=your_verified_sender@example.com
ALERT_FROM_NAME=MetalPulse Alerts

# Optional: worker interval (default 15 minutes)
ALERT_CHECK_INTERVAL_MS=900000
```

### Development

```bash
# Run scraper (metals history)
npm run dev:scraper

# Run API server (metals + ETFs + frontend static)
npm run dev:api

# Run alerts worker
npm run dev:worker

# All together (recommended for local dev)
npm run dev:alerts:stack
```

- Frontend runs on `http://127.0.0.1:8080`
- API on `http://127.0.0.1:8788`
- Scraper on `http://127.0.0.1:8787`

### Build

```bash
npm run build
```

Outputs to `dist/`. The API server serves the static frontend with SPA fallback.

## Deployment (Railway)

### Services

1. **API Service** (public domain)
   - Start command: `npm run start:api`
   - Environment: `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`, `TWELVEDATA_API_KEY`

2. **Worker Service** (private, no domain)
   - Start command: `npm run start:worker`
   - Environment: `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`, `SENDGRID_API_KEY`, `ALERT_FROM_EMAIL`, `ALERT_FROM_NAME`

3. **(Optional) Scraper Service** (can be merged into API)
   - Start command: `npm run start:scraper`

### Notes

- Railway injects `PORT`; the API server listens on it.
- Worker does **not** need a public domain; it runs privately.
- Ensure `TWELVEDATA_API_KEY` is set on the API service to fix ETF 503 errors.
- Ensure `SENDGRID_API_KEY` and `ALERT_FROM_EMAIL` are set on Worker for email alerts.

## API Endpoints

| Method | Path | Description |
|--------|-------|-------------|
| GET    | `/health` | Health check |
| GET    | `/api/metals` | Live spot prices + recent daily high/low |
| GET    | `/api/metals/:symbol/history?days=N` | Historical daily closes (Stooq) |
| GET    | `/api/etfs` | Live ETF quotes (TwelveData) |
| POST   | `/api/alerts` | Create a price alert |
| GET    | `/api/alerts` | List alerts (if needed in future) |

## Architecture Notes

- **Worker** evaluates crossing: `null → true` or `false → true` triggers an email via SendGrid.
- **Price fetching**: Stooq is tried first to save API quota; TwelveData fills gaps only when needed.
- **PWA**: Installable on phones; manifest generated by `vite-plugin-pwa`.
